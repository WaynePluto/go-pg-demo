# 相关上下文

- internal/modules/iacc 目录下是身份与访问控制相关的代码，主要有用户、角色、权限三个主数据表，对于的type.go文件中定义了数据库表实体和对应接口的输入输出结构体，handler.go文件中定义了实现的接口对应的处理逻辑。

- internal/middlewares/auth.go 中定义了身份认证的中间件逻辑，主要是通过解析请求头中的token来获取用户信息，并进行权限校验。

# 需求

在internal/modules/iacc/auth目录下实现三个接口，分别是用户登录、刷新token、获取用户详细信息。
- 因为是数据库中并不存在 auth相关的数据表，所以对应要实现的type.go中只有接口的输入输出结构体定义，没有数据库表实体定义。
- 参考`internal/modules/template`目录下的CURD接口模板，要求实现的接口要遵守`promot/rules/sqlx.md`内的规则。
- 获取用户的详细信息这个接口，要返回用户的基本信息、角色列表、权限列表等数据。
- 只要实现接口的type.go和handler.go文件内容即可。
